# App Release Privacy & Ad Checklist (AdMob)

이 문서는 AdMob 적용 앱을 스토어 배포할 때 필수로 준비해야 하는 항목을 정리한 체크리스트입니다.

## 1) 법적 문서

- [ ] 개인정보처리방침(Privacy Policy) 공개 URL 준비
- [ ] 서비스 이용약관(선택) URL 준비
- [ ] 앱 내 설정 화면에서 개인정보처리방침 접근 경로 제공

권장 포함 내용:
- 수집 데이터 항목(광고 식별자, 기기 정보, 앱 사용 이벤트 등)
- 수집 목적(광고 제공, 성능 분석, 부정 사용 방지 등)
- 제3자 제공/처리위탁(예: Google AdMob)
- 보관 기간, 파기 정책
- 사용자 권리(동의 철회, 문의 채널)

## 2) 동의/거부 UX

- [ ] 현재 정책 고정: 개인화 광고 미사용, non-personalized ads만 요청
- [ ] 앱 설정에서 처리방침 안내 및 정책(비개인화 광고) 명시
- [ ] 추후 개인화 광고 도입 시 CMP/동의 철회 UX 추가 설계

iOS 추가:
- [ ] ATT(AppTrackingTransparency) 권한 문구 준비
- [ ] ATT 미동의 시 추적 기반 개인화 비활성화

## 3) 스토어 메타데이터

### Google Play
- [ ] Data Safety 설문 정확히 작성
- [ ] 광고 SDK 사용(AdMob) 항목 반영
- [ ] 개인정보처리방침 URL 등록

### Apple App Store
- [ ] App Privacy(데이터 수집/추적) 항목 작성
- [ ] ATT 사용 시 추적 관련 답변 일치 확인
- [ ] 개인정보처리방침 URL 등록

## 4) AdMob 운영 준비

- [ ] 앱 등록(Android/iOS)
- [ ] 광고 단위 생성(Interstitial, Rewarded)
- [ ] 테스트 광고 단위/테스트 디바이스 검증
- [ ] 운영 광고 단위 분리

## 5) 런타임 정책(현재 프로젝트 기준)

현재 프로젝트는 floating 유저 단일 정책으로 동작합니다.

- 전면광고(클리어):
  - [ ] 정책 캡 적용(세션/일/쿨다운)
- 보상형 광고:
  - [ ] 클리어 X2 보상
  - [ ] 힌트 언락(골드 부족 시)
- [ ] 광고 실패 시 대체 UX 제공(기본 보상 유지, 안내 문구)

## 6) QA 시나리오

- [ ] 개인화 ON/OFF 전환 시 광고 요청 플래그 반영 확인
- [ ] 클리어 전면광고 노출 주기/캡 정상 동작
- [ ] Rewarded 완료 콜백 시에만 보상 지급
- [ ] 네트워크 실패/광고 로드 실패 시 게임 진행 막힘 없음
- [ ] 게스트/연동 계정 모두 저장 상태 정상

## 7) 운영 가드레일

- [ ] 원격 설정으로 광고 ON/OFF 가능
- [ ] 초반 배포는 광고 OFF 후 점진 ON 권장
- [ ] 이벤트 로깅(노출/클릭/완료/실패)으로 모니터링

## 참고

- 본 문서는 법률 자문이 아니며, 실제 배포 전 지역별 법규 및 스토어 정책 최신 버전을 검토해야 합니다.
